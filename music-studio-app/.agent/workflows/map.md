---
description: Analyze the codebase and generate/update ARCHITECTURE.md.
---

1. **Explore**:
   - Run `list_dir` to see the file structure.
   - Read `package.json`, `README.md`, or other config files to understand dependencies and entry points.

2. **Analyze**:
   - Identify key components, modules, and their relationships.
   - Identify the tech stack (Frameworks, Libraries, Tools).

3. **Document**:
   - Create or update `.gsd/ARCHITECTURE.md`.
   - **Structure**:
     - **Overview**: High-level system description.
     - **Directory Structure**: Key directories and their purpose.
     - **Components**: Major modules and how they interact.
     - **Data Flow**: How data moves through the system.

4. **Inventory**:
   - Update `.gsd/STACK.md` with the technologies found.
